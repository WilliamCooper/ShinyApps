Instructions for the “Options for Vertical Wind” Tab

This tab has options for the addition of two variables to netCDF files, WIF (resulting from high-pass filtering of the vertical wind to eliminate undesirable offsets) and WIY (produced by the complementary-filter representation of angle of attack). The new variable for angle of attack, AKY, is also added to the files.

To use this tab, select the checkboxes indicating which variables are to be added to the data files and select the “Run WI Options” button. You can also use the “Next” button to process the next unprocessed flight, or the “All” button to process all flights for a selected project. The resulting files are named with a “X” suffix following this pattern: “CSETrf01X.nc”. Selecting an already processed flight will reprocess that flight; selecting “All” will process only flights that have not yet been processed.

The WIF calculation will work for all projects, but for projects preceding DEEPWAVE the WIY solution will produce unsatisfactory results so that should not be used unless new calibration coefficients for AKY are determined. The calibration coefficients for AKY were determined from a combination of data from DEEPWAVE, CSET, and ORCAS. This is documented in the Kalman Filter technical note and in a more extensive memo referenced in that technical note.
