Toward reproducible analysis:

An advantage of the structure provided by the combination of R, Rstudio, and knitr is that the text generating a report and the code generating the results and plots can be combined into a single file that, along with the data used, can then allow someone else to reproduce the results or modify them with new data or a new analysis approach. As an example, see the program file and resulting memo included in the ’RSessions/Session1’ directory as CONTRASTcalAOA.Rnw and CONTRASTcalAOA.pdf. I’ll use those next to give an overview of how this can be used.

Characteristics of this file to note:

  1. Text is intermixed with R code in this file. R code ’chunks’ begin with a header line <﻿<chunk-name>﻿>= and end with @. Results from the R code can be incorporated into subsequent parts of the text.

  2. The chunks provide a way to structure your program and to skip to appropriate sections of the code.

  3. The text will have references to the R results in the form of ’\Sexpr{x}’ where x is an R variable. (Sexpr is a remnant from R’s origin in S; Sexpr = S expression.)

  4. If a figure is generated in an R chunk, it appears in the text document after the place where the R chunk appears.

  5. I have emphasized reproducibility and have included a section on how to use the file with other data and a table on reproducibility at the end of the memo. That table contains:

    1. the project name

    2. the location of the source file

    3. the name of a zip-format archive containing all the information needed to repeat the production of the memo

    4. the original data used. (Only the subset of the data used here is saved in the archive to keep the size of these archive files small.)

    5. the archive location on github, which anyone can access

    This will be the subject of a later session in this series, but the goal of facilitating reproducibility can be motivation to continue along this path.
