R Objects

with focus on the data.frame

vector

The fundamental object we will deal with most often is the vector. This is an indexed set of quantities, which can be numeric (double, integer), logical, character, or some other less common types. Vectors are ’atomic’ if they consist all of the same type; ’generic’ vectors can have mixed types, but I won’t deal much with them. Vectors may have only one element, which would be called a scalar in other languages, but scalars in R are just vectors with one element. Vectors have a length property [try length(x)] but not a dimension property: Try the following in the Rstudio console or an R session:

x <- 2 ## assign 2 to x, note the assignment operator “<-” and “#” for comments
print (x[1])

## [1] 2 ## statements beginning with ##[n] here are R output

length(x)

## [1] 1

dim(x)

## NULL

list

A list resembles an atomic vector but its elements can have mixed types. Lists also have additional ways of selecting elements, e.g., by character name. Lists are generic vectors, but when I talk about vectors I’ll almost always be talking about atomic vectors. Lists are addressed in ways that are different from methods for vectors. You will have to study this later; for now, just be aware that lists are available and the list is the most flexible storage object.

array (including matrix)

A special vector-like object is the array or matrix, which is like a vector but with assigned dimensions so that, in the two-dimensional or matrix case, the vector is a vector of columns in a matrix. The dimension can be single-valued, but more often there are at least two dimensions. R convention is column-major format (like FORTRAN but opposite to C); i.e., the row index represents adjacent values in the underlying vector, so items that lie vertically in the conventional display of a matrix (for example z[1,1] and z[2,1]) are adjacent elements in the underlying vector. The first index in a two-dimensional array can be thought of as specifying the row and the second the column; all rows in a particular column are contiguous in the underlying vector. You can construct an array from a vector by assigning it a dimension via ’dim(x) <- c(dim1, dim2)’ as in the following exercise, which also illustrates the column-major structure and the effect of transposing the array using t().
